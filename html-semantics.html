HTML Semantics

HTML semantics means using HTML elements that have a clear and specific meaning about the type of content they hold. Instead of just telling the browser how something should look, semantic elements describe what the content is.

For example, <header> is meant for the top section of a page or section, <nav> is for navigation links, <article> is for a standalone piece of content, and <footer> is for the bottom section of a page. These names make your HTML easier to read and help search engines and assistive technologies understand your content better.

Here is an example of what the <body> would contain when using HTML semantics:

<header>
  <h1>My Blog</h1>
</header>

<nav>
  <a href="home.html">Home</a>
  <a href="about.html">About</a>
</nav>

<article>
  <h2>My First Post</h2>
  <p>This is the content of my first blog post.</p>
</article>

<footer>
  <p>Copyright 2025 My Blog</p>
</footer>
